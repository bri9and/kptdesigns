All checks performed against `http://localhost:3000` (dev server) and engine source under `src/app/mockup/<slug>/`.

## Methodology

Per-concept checks:

1. **200?** — `curl` against the dev route returns HTTP 200.
2. **headline-match?** — Brief's hero headline appears in the engine source. Comparison normalizes HTML entities (`&rsquo;`, `&mdash;`), smart quotes, and ASCII dashes; it splits the headline by sentence/clause boundaries and accepts an exact-substring hit, all-fragments-present, or ≥70% content-token overlap (since headlines are routinely split across spans/`<br />` tags or rendered as data instead of inline text).
3. **wrapper?** — `page.tsx` references `PageApproach`.
4. **sections>=3?** — Engine emits ≥3 `<section>` tags (or other top-level block elements where higher).
5. **hover/focus?** — Engine source contains at least one `:hover` or `:focus` selector (inline styled-jsx or CSS module).
6. **palette?** — At least 2 of the 3 hex codes from the concept-matrix palette appear in the engine source. Match accepts the literal hex (`#C04D2A`) or its `rgb()`/`rgba()` decimal encoding (e.g. `rgba(192,77,42,0.22)`), since several engines define accent stops via `rgba` for alpha control.
7. **no-fetch?** — Engine + page contain zero references to `fetch(`, `useSWR`, `axios`, `@clerk`, or `@supabase` (mockups must be static).
