# 50-Mockup Concept Catalog Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 50 visually-distinct full-page concept mockups (`/mockup/v23-...` through `/mockup/v72-...`) to the kptdesigns picker, plus a 5-gate review workflow that prevents sameness and quality drift across the catalog.

**Architecture:** A 6-role subagent team orchestrated by the Director (the conversation). Five sequenced gates: matrix → briefs → build → QA/critique → final walk. Each gate has a deliverable the Director reviews before the next gate opens. Build squad parallelizes ~5 concepts per agent across ~10 agents.

**Tech Stack:** Next.js 16 App Router, React Server Components for `page.tsx` entry points, client components inside `_engine/<Name>.tsx`, inline `<style>` tags following the existing cosmic-page convention. Mandatory `<PageApproach>` wrapper from `src/app/mockup/_lib/PageApproach.tsx`. Tailwind not assumed (each engine is self-contained CSS); framer-motion is allowed where motion is core to the concept.

**Spec:** `docs/superpowers/specs/2026-04-28-kpt-50-mockups-design.md`

### Code (Gate 3 build output)

| Path pattern | Count | What |
|---|---|---|
| `src/app/mockup/v<NN>-<slug>/page.tsx` | 50 | Server entry, wraps engine in `<PageApproach>` |
| `src/app/mockup/v<NN>-<slug>/_engine/<Name>.tsx` | 50 | Client engine, the actual page |

### Branch + commits

All work lands on `feat/mockup-playground` (the current branch). One commit per Gate
deliverable; build commits at task batch boundaries (every 5 mockups). Final deploy via
`vercel --prod` after Gate 5.
