# Microcopy Palette — V5 through V20

Each design entry contains: tone designation, button labels, form placeholders, error/empty/loading states, tooltips, and toast messages. Copy stays inside the visual language of each design brief. Lining figures (0–9) are used for all numerals.

---

## Cross-design Copy Rules

These rules are invariant across all 16 designs. Override them only with explicit sign-off.

1. **Primary CTA always points to `/start`.** Label must be imperative and active — "Start", "Begin", "Launch", "Get Started", "Build Mine", "Transmit", "Open Session". Never "Submit", "Go", "Click Here", or "Learn More."
2. **Pricing CTA always points to `/pricing`.** Label varies by design voice but must be unambiguous: "Pricing", "See Pricing", "View Rates", "Rate Card", "Tariff Schedule", etc. Never "Buy Now" or "Upgrade."
3. **Numerals use lining figures everywhere** — in body copy, stat callouts, pricing, date stamps, coordinates. No oldstyle figures in running copy (Fraunces variable: use `font-variant-numeric: lining-nums`).
4. **Error messages name the field and suggest a fix.** Never "An error occurred." Always "Email address is missing — please add one."
5. **Empty state copy points toward an action.** Never just "No results." Always "No results yet — [actionable prompt]."
6. **Loading states are progress-positive.** Never "Loading…" alone. Always imply work happening: "Fetching your sites…", "Scanning portfolio…", etc.
7. **Success toasts are brief and warm.** One sentence. No exclamation marks on designs with dry/deadpan/literary tone; exclamation marks permitted on warm/punk/friendly tones.
8. **Warning toasts do not block.** They inform; they never prevent proceeding.
9. **Tooltip copy is sub-12-words.** If it takes more than 12 words to explain a feature, that feature needs a label, not a tooltip.
10. **"KPT Designs" is always two words, always title-case.** Never "KPTDesigns", "kpt designs", "KPT designs."
