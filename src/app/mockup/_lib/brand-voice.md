## 6. Process Step Copy

Four stages. Two voice variants per stage.

### Stage 1 — Discovery

**Variant A (warm/human):**
We start by listening. One conversation is usually enough to understand your audience, your constraints, and what you have tried before. We come in with questions, not a template.

**Variant B (technical/precise):**
Discovery: requirements capture, content audit, technical constraints review. Deliverable: a signed scope document with stack selection, timeline, and acceptance criteria.

---

### Stage 2 — Build

**Variant A (warm/human):**
We build in the open. You get a staging URL on day one and a new deploy every time something significant changes. Nothing is hidden until it is finished.

**Variant B (technical/precise):**
Build: feature branches merged to staging on Vercel preview deployments. TypeScript strict mode enforced. Core Web Vitals baseline measured at each merge. Client has read access to the repository throughout.

---

### Stage 3 — Review

**Variant A (warm/human):**
Review is a conversation, not a presentation. We walk through every section together, take notes in real time, and ship the changes the same day when possible.

**Variant B (technical/precise):**
Review: client walkthrough on staging, annotated feedback captured in GitHub Issues, change set scoped and prioritized, revised deploy within the agreed turnaround window.

---

### Stage 4 — Delivery

**Variant A (warm/human):**
When we hand it over, it is actually yours. Domain, hosting, and code — all transferred, all documented. We stay available, but you do not need us to keep the lights on.

**Variant B (technical/precise):**
Delivery: DNS cutover, production deploy, repository ownership transferred to client's GitHub org, environment variables documented. Hosting config handed off or managed on retainer — client's choice.

## 10. Voice Rules

### DO

- Write short sentences. Break long thoughts in two.
- Use specific numbers: "47 sites," "7 to 90 days," "sub-50ms."
- Let the fact carry the weight. Skip the adjective.
- Write in plain English. If you need a glossary entry, simplify.
- Understate. The work speaks; the copy should not oversell.
- Use "we" freely. It is a real team, not a corporate voice.
- Refer to project handoffs as ownership transfers — because that is what they are.
- Mention KPT Agents by name when referencing AI phone capability. It is a real company.

### DON'T

- No exclamation marks. Ever.
- No "we're excited to," "thrilled to," "proud to."
- No "synergy," "ecosystem," "holistic," "turnkey," "cutting-edge."
- No "AI-powered." Say "AI agents" or name the capability directly.
- No "solutions" as a noun.
- No rhetorical questions in hero copy.
- No ellipsis for emphasis… it reads as uncertain.
- No emojis anywhere in production copy.
- Do not pad: "In today's fast-paced digital landscape" earns an immediate delete.
- Do not claim the superlative: "best," "most," "leading" without a citation.

## 11. Tone by Design Family

Copy must stay aligned with each design family's visual register. Adjust diction, sentence rhythm, and formality accordingly.

| Family | Designs | Tone Directive |
|--------|---------|----------------|
| **Cosmic / Space** | V2 Cosmos, V5 Tunnel, V17 Nostromo | Awe without hype. Short declarative sentences with long pauses. Lean toward the poetic and the vast. Numbers as coordinates, not marketing. |
| **Editorial / Print** | V3 Editorial, V8 Codex, V10 Atelier, V12 Specimen, V16 Broadsheet, V19 Risograph | Restrained. Authoritative. Type-led rhythm — short clauses, deliberate white space. Sound like a considered publication, not a pitch deck. |
| **Terminal / Developer** | V4 Terminal, V20 Operator | Dry. Precise. Functional. Use technical vocabulary correctly. No warmth that would feel misplaced in a shell prompt. |
| **Architectural / Structural** | V6 Strata, V11 Architectural, V15 Atlas | Measured. Structural. Let the layout carry meaning; copy is sparse annotation, not narration. |
| **Motion / Zoom** | V7 Recursive | Economical. Each word must survive the zoom. Avoid phrases that read poorly at large scale. |
| **Conversational / Human** | V13 Conversation, V18 Receipt | Warm but not saccharine. Short questions are acceptable here. Read aloud; if it sounds like a person, keep it. |
| **Retro / Analog** | V9 Ticker, V14 Cassette | Voice matches the era being referenced — laconic, slightly worn, confident in an understated way. No irony that undercuts the brand. |
